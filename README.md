#  經典打地鼠遊戲與回饋表單

##  專案簡介
本專案是結合經典打地鼠遊戲和意見回饋表單的互動網頁應用程式。遊戲區提供計時和計分功能，模擬打地鼠的遊戲體驗；遊戲結束後，使用者可查看分數並選擇填寫表單提供建議。系統整合 Bootstrap 響應式設計、原生表單驗證及 JavaScript DOM 操作。

---

##  使用技術

- HTML5 & CSS3：專案結構與基本樣式。
- Bootstrap 5：用於快速建立響應式佈局、表單元件、按鈕、Modal 視窗等。
- JavaScript (ES6+)：主要的遊戲邏輯、計時器、分數計算、DOM 元素操作等。
- JS-Confetti (CDN)：用於遊戲結束時的分數慶祝動畫特效。
- Constraint Validation API：配合 HTML5 內建屬性及 JS 邏輯，實現表單即時驗證。

---

##  功能特色

### I. 遊戲核心功能 (Whack-A-Mole Game)

1.  時間與分數系統
    - 30 秒計時機制：遊戲時間設定為 30 秒，即時顯示剩餘時間。
    - 即時計分：成功擊中地鼠即時增加分數。
2.  地鼠隨機彈出
    - 地鼠會從 9 個洞中隨機彈出，並避免連續出現在同一洞。
    - 地鼠出現和消失的時間採隨機延遲，增加遊戲變化性。
3.  擊中特效
    - 擊中地鼠後，地鼠圖片會替換成被打暈的狀態（'mole-dizzy' 樣式），並有短暫的 'hit' 動畫和發光效果。
4.  倒數警告
    - 遊戲最後 5 秒會觸發倒數警告提示 ('timer-warning' 樣式)，以脈衝動畫提醒時間將至。

### II. 結果與回饋表單功能 (Feedback Form)

1.  遊戲結果顯示
    - 遊戲結束後，自動切換至結果區，顯示最終得分，並播放彩帶/愛心 (Confetti) 慶祝動畫。
    - 最終分數設有獨立的 CSS 慶祝動畫 (`score-highlight`)。
2.  表單即時驗證
    - 利用 Constraint Validation API，對必填欄位 (如**滿意度**) 進行原生驗證。
    - 客製化信箱格式驗證：JavaScript 檢查信箱必須是 `@gmail.com` 或 `@o365.tku.edu.tw` 格式。
3.  防止重複提交
    - 表單提交成功後，**提交按鈕會被禁用**，防止使用者重複送出資料。
4.  成功提示
    - 表單提交後，會跳出一個 **Bootstrap Modal 視窗** 提示使用者提交成功。
